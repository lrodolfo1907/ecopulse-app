# Contributing to Ecopulse App

Thank you for your interest in contributing to the Ecopulse App! We welcome contributions from everyone. Please take a moment to read through the guidelines below.

## Contribution Guidelines

1. **Fork the repository**: Start by forking the repository to your own GitHub account.
2. **Create a new branch**: Create a new branch for your feature or bugfix. Use descriptive names for your branches.
   
   Example: `git checkout -b feature/new-feature`
3. **Make your changes**: Implement your feature or bug fix.
4. **Write tests**: Ensure that your changes are covered by tests. Run the test suite to verify.
5. **Commit your changes**: Commit your changes with descriptive messages.
   
   Example: `git commit -m 'Add new feature that does amazing things'`
6. **Push to your fork**: Push your changes back to your forked repository.
   
   Example: `git push origin feature/new-feature`
7. **Create a pull request**: Go to the original repository and create a pull request. Provide a clear description of your changes and why they are necessary.

## Development Workflow

- **Clone the repository**: Use `git clone <repo-url>` to clone the repository to your local machine.
- **Set up the project**: Follow the instructions in the `README.md` to set up the project on your local machine.
- **Start development**: Work on your feature or bug fix in your branch.
- **Pull the latest changes**: Before creating a pull request, make sure to pull the latest changes from the main branch to avoid merge conflicts.

## Code of Conduct

We expect everyone to follow our code of conduct. Please be respectful and considerate to others.

## Thank You!

We appreciate your interest in contributing to the Ecopulse App! Your contributions help make this project better.
